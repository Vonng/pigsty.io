草稿国产数据库到底行不行？

By 冯若航（@Vonng）| 微信公众号 | 2024年04月25日
Categories:
  • Database
Tags:
  • National

总有朋友问我，国产数据库到底能不能打？说实话，是个得罪人的问题。所以我们不妨试试用数据说话 —— 希望本文提供的图表，能够帮助读者了解数据库生态格局，并建立更为准确的比例感认知。

数据源与研究方法

关于数据库流行度 & 影响力这个问题，我认为有三份数据可以作为参考：StackOverflow 全球开发者调研，DB-Engine 数据库流行度排行榜，以及墨天轮国产数据库排行榜。

其中，最有参考价值的是 StackOverflow 2017 - 2023 年的全球开发者问卷调研 —— 样本调查获取的第一手数据具有高度的可信度与说服力，并且具有极好的 横向可比性（不同数据库之间对比）；连续七年的调查结果也有着足够的 纵向可比性 （某个数据库和自己过去的历史相比）。

其次是 DB-Engine 数据库流行度排行榜， DB-Engine 属于综合性热搜指数，将 Google, Bing, Google Trends，StackOverflow，DBA Stack Exchange，Indeed, Simply Hired， LinkedIn，Twitter 上的间接数据合成了一个热搜指数。

这份指数有着极好的 纵向可比性 —— 我们可以轻松看出在同一个标准下，某个数据库的“热度”走势，是更流行了还是更过气了。但在 横向可比性 上表现不佳 —— 例如你没办法细分用户搜索的目的，所以在水平对比不同数据库时，只能作为一个模糊的参考 —— 起码在数量级上的准确性还是可以认可的。

第三份数据是墨天轮的“国产数据库排行榜”，这份榜单收录了 287 个国产数据库，用类似于 DB-Engine 的方式并给出了它们的“热度”，可以作为一个关于国产数据库的专项补充参考。这里我们简单认为 —— 收录在这里的数据库，就算广义上的国产数据库了 —— 尽管这些数据库团队，不见得自我认知并喜欢这个标签。

在 DB-Engine 收录的全球 478 款数据库中，可以找到 46 款列入墨天轮国产数据库名单的产品。

有了这三份数据，我们就可以尝试回答这个问题 —— 国产数据库在国际上的流行度与影响力到底是什么水平？

卑微的国产数据库

不难发现 TiDB 是唯一一个，同时出现在这三个榜单里的知名数据库，因此可以作为一个很好的 锚点。在 StackOverflow 2023 调研中，TiDB 作为最后一名，首次出现在数据库流行度榜单里，也是唯一入选的 “国产数据库”。

TiDB 的使用率为 0.20%，与排名第一的 PostgreSQL (45.55%) 和排名第二的 MySQL (41.09%) 相比，流行度相差了大约 两三百倍。我们可以从第二份 DB-Engine 上的数据交叉印证这一点 —— TiDB 在 DB-Engine 上的评分是国产数据库中最高的 —— 在2024年4月份，为 5.14 分。与排名前四的关系数据库四大天王相比，也是小几百倍的差距。

在墨天轮国产数据库排名中，TiDB 曾经长时间占据了榜首的位置，尽管最近两年前面加塞了 OceanBase， PolarDB，openGauss 三个数据库，但它还在第一梯队里，称其为标杆也没有什么大问题。

从 DB-Engine 的热度趋势不难看出，国产数据库从 2017 - 2020 年开始起势，从 2021 年进入高潮，在 23年5月进入平台期，从今年年初开始，出现掉头过气的趋势。这和许多业内专家的判断一致 —— 2024 年，国产数据库进入洗牌清算期 —— 大量数据库公司将倒闭破产或被合并收编。

这里一个比较明显的例外是 TiDB ，TiDB 也许从替代 MySQL 中获得了一些额外增长动能。它确实算这堆 “国产数据库” 中最能打的了。虽然 TiDB 主打一个出海，自己可能并不高兴被称为 ”国产数据库“ —— 但这正好使得 TiDB 称为对比全球数据库与国产数据库的最佳参考锚点。

因此如果我们以 TiDB 作为参考锚点，可以得出一个挺有趣的结论：国产数据库看上去人才济济，群英荟萃，但即使是最能打的国产数据库，流行度与影响力也不及 PostgreSQL 的百分之一……。

而更过分的是，把整个国产数据库的热度分数全加起来，也甚至还达不到 PostgreSQL 的零头 …… 微不足道到完全可以合并入 “其他” 统计项中。整个 “国产数据库” 运动 —— 我真的不确定 —— 它否能在数据库的历史中留痕。

过气中的数据库们

但其实，数据库过气并非国产数据库独有的问题 —— 绝大多数的数据库其实都正在过气中。DB-Engine 过去12 年中的流行度数据趋势可以揭示这一点 —— 尽管 DB-Engine 热度指标的的横向可比性很一般，但纵向可比性还是很不错的 —— 因此在判断流行 & 过气趋势上仍然有很大的参考价值 —— 我们可以对图表做一个加工处理 —— 以某一年为零点，来看热度分数从此刻起的变化，从而看出那些数据库正在繁荣发展，哪些数据库正在落伍过气。

如果我们将目光聚焦在最近三年，不难发现在所有数据库中，只有 PostgreSQL 与 Snowflake 的流行度有显著增长。而最大的输家是 SQL Server，Oracle，MySQL，与 MongoDB …… 。分析数仓类组件（广义上的数据库）在最近三年有少量增长，而绝大部分其他数据库都处在过气通道中。

如果我们以 DB-Engine 最早有记录的 2012-11 作为参考零点，那么 PostgreSQL 是过去 12 年中数据库领域的最大赢家；而最大的输家依然是 SQL Server，Oracle，MySQL 御三家关系型数据库。但不难发现，NoSQL 运动的兴起，让 MongoDB ，ElasticSearch，Redis 在 2012 - 2022 互联网黄金十年中获得了可观的增长，但这个增长的势头在最近几年已经结束了，并进入过气下降通道中，进入吃存量老本的状态。

接下来，我们将这些数据库分为三类：SQL，NoSQL，以及 NewSQL，看看这些数据库到底是更流行了，还是开始过气了。

关系型：此消彼长

SQL关系型数据库一直很重要，但头部关系型数据库除了 PostgreSQL 之外，都在走下坡路。

从 DB-Engine 数据上不难看出，头部三家关系型数据库已经走了十几年下坡路了，而除了 PostgreSQL 之外的几乎所有头部关系型数据库，在最近三年都在走下坡路。MariaDB 作为 MySQL 的平替，在荣枯线上下波动，倒是一些新兴的 SQL 分析数仓，在最近几年有一些增长。

当然还是要在此提醒大家注意 —— 热度类指标仅适合说明趋势，而横向可比性很差 —— 难以说明热度在同一个数量级内的数据库具体谁更流行。横向可比的流行度，当参考最右 StackOverflow 开发者问卷调研数据。

从主要关系型数据库的流行度变化，我们也可以看到另一个趋势 —— 即开源数据库对闭源商业数据库的碾压态势 —— 开源的数据库如 PostgreSQL，MariaDB，SQLite 都有增长，商业的数据库 Oracle，MSSQL，DB2，Access 都大幅衰退，唯一的特例是开源 MySQL —— 这也许是因为它是由单一商业主体 Oracle 所掌控的 “开源”，被锁死了发展天花板，并在挤牙膏式更新中逐渐失去了竞争力。

NoSQL：昨日辉煌

NoSQL 曾经火过，并留下了三个不错的数据组件： Redis，Mongo，ElasticSearch —— 很巧的是，它们都修改为 “非开源协议” 并正在和公有云厂商掰扯搏斗。这些 NoSQL 组件还有不错的存量家底可以霍霍，它们会经历一个缓慢但确定的过气过程。

这里特别值得一提的是 MongoDB，作为率先提出文档数据库这一新模型的创新型产品，MongoDB 在12 - 22 这互联网黄金十年吃到了仅次于 PostgreSQL 的红利，Mongo 与 PG 在 DB-Engine 中画出了最显眼的两条增长曲线。

但不幸地是，PostgreSQL 在 2012 ～ 2014 年 9.2～9.4 版本中同样引入了完整的 JSON 文档特性支持，与高效的 GIN 索引。在功能上完全替代了 MongoDB，也能够同时使用关系模型；更重要的是，在性能，功能，正确性上都显著优于 MongoDB —— 因此压制住了 MongoDB 的增长势头。MongoDB 的增长动能持续到了 2021 年，随即出现了显著的转折，开始掉头过气 —— 这也许是受到了 PostgreSQL 替代，与公有云“开源替代” 如 DocumentDB 的双重夹击导致的。

作为 NoSQL 运动的扛旗者，MongoDB，Redis，ElasticSearch 曾经辉煌过。而它们的过气，也标志着 NoSQL 运动将告一段落。

NewSQL：穷途末路

NewSQL 运动，即所谓的新一代分布式数据库。如果说 NoSQL 起码辉煌过，那么可以说 NewSQL 还没辉煌就已经熄火了。“分布式数据库” 在国内营销炒作的非常火热，以至于大家好像把它当作一个可以与 “集中式数据库” 分庭抗礼的数据库品类来看待。但如果我们深入研究就不难发现 —— 这其实只是一个非常冷门的数据库小众领域。

一些 NoSQL 组件的流行度还能和 PostgreSQL 放到同一个坐标图中而不显突兀，而所有 NewSQL 玩家加起来的流行度分数也比不上 PostgreSQL 的零头 —— 和“国产数据库”一样。

总的来说，硬件遵循摩尔定律飞速进行指数级地降本增效，这让 NewSQL 的 “海量” 价值主张雪上加霜，并让 分布式数据库几乎成为了一个伪需求，这个品类和 “国产数据库”一样无足轻重。私以为，在这个领域也许能剩下两个主要玩家 CockroachDB 与 TiDB，分别作为 PG & MySQL 的衍生兼容品，在 NewSQL 螺丝壳里打局部代理人战争。

重走 Linux 的道路

几份数据为我们揭示出数据库领域的基本格局 —— 除了 PostgreSQL 之外的主要数据库都在过气中，无论是 SQL，NoSQL，还是 NewSQL 。这确实抛出了一个有趣的问题，让人想问 —— 为什么？。

对于这个问题，我在 《PostgreSQL 正在吞噬数据库世界》中提出了一种简单的解释：PostgreSQL 正在凭借其强大的扩展插件，内化吞噬整个数据库世界。根据奥卡姆剃刀原理 —— 最简单的解释往往也最接近真相。

Linux 内核 一统服务器操作系统天下后，曾经的同台竞争者 BSD，Solaris，Unix 都成为了时代的注脚。而我们正在目睹同样的事情在数据库领域发生 —— 在这个时代里，想发明新的实用数据库内核，基本上等于堂吉柯德撞风车。今天我们看 Linux 操作系统内核 —— 尽管市面上有这么多的 Linux 操作系统发行版，但大家都选择使用同样的 Linux 内核，吃饱了撑着魔改内核属于没有困难创造困难也要上，会被业界当成山炮看待。

同理，数据库内核本身已经不再是主要矛盾 —— 看上去 PostgreSQL 和 MySQL 生态还要搏杀一阵才能分出胜负。但在远见者眼中，这场纷争早就已经尘埃落定了。竞争的焦点已经转移到了 PostgreSQL 生态内部 —— 如果 PostgreSQL 注定成为数据库领域的 Linux 内核，那么谁会成为 Postgres 的 Debian / Ubuntu / Suse / RedHat ？

做内核的厂商不温不火，MariaDB 作为 MySQL 的亲爹 Fork 甚至都已经濒临退市，而白嫖内核自己做服务与扩展卖 RDS 的 AWS 可以赚的钵满盆翻，甚至凭借这种模式一路干到了数据库市场份额的榜首。

尝试在本地提供更好、更便宜、 RDS for PostgreSQL 的 Pigsty 对这种模式提出了挑战，同时还有十几款尝试用 云原生方式解决 RDS 本地化挑战的 Kubernetes Operator 正在摩拳擦掌，跃跃欲试。

数据库领域正在从寒武纪大爆发走向侏罗纪大灭绝，在这一过程中，1% 的种子将会继承 99% 的未来，并演化出新的生态与规则。我建议数据库用户站在未来与希望的一侧 —— 拿好船票，让我们扬帆起航吧。
